Byron is built to take source documents exactly as they arrive from clients and brokerages. In most cases you can drag a package straight into a filing without sorting, renaming, or converting anything. This page covers which file formats Byron accepts and what to do when a file doesn't fit the supported list.

<Note>
  While Byron currently supports PDF and Excel files only, we’re actively working on adding support for image files, Word documents, and ZIP archives
</Note>

Byron accepts two file types today.

<AccordionGroup>
  <Accordion title="PDF">
    **PDF (**`.pdf`**)** is the primary format. Every tax document Byron processes — W-2s, 1099s, K-1s, K-3s, brokerage statements, Schedule C profit-and-loss statements, and the rest — is accepted as a PDF. 

    Byron handles real-world PDF quality, not just clean exports, which means both digital PDFs and scanned PDFs work. A scanned PDF from a client's paper stack, or a page with coffee stains or highlighter marks, will still be processed. Byron uses AI models for extraction rather than traditional OCR, which means it's tolerant of imperfect scans, skewed pages, and low-resolution captures.
  </Accordion>

  <Accordion title="Excel">
    **Excel (**`.xlsx`**)** is accepted for tabular source data that arrives as a spreadsheet rather than a printed form — Schedule C working files, trial balances, and similar workbooks.
  </Accordion>
</AccordionGroup>

Upload is drag-and-drop from anywhere on the filing workspace, or by clicking **Upload** and selecting files from your computer.

## What's not yet supported

These formats are on the roadmap but need to be converted before upload today:

- **Image files** (`.jpg`, `.jpeg`, `.png`, `.heic`) — _Coming soon._ For now, save to PDF first. On iPhone, the Files app can convert an image to PDF via the share sheet; on desktop, most operating systems can "Print to PDF" from an image preview.
- **Word documents** (`.doc`, `.docx`) — _Coming soon._ For now, print to PDF before uploading.
- **Zip archives** (`.zip`) — _Coming soon._ For now, extract the archive and upload the individual files inside.

Byron's uploader doesn't accept a folder as a single drag target — upload the individual files inside the folder.

If you're not sure whether a specific file will work, upload it. If Byron can't process it, the rejection reason appears on the file tile in the filing workspace so you can fix and re-upload.

## Multiple files at once

You can drop an entire client package in one go. Byron auto-classifies each file — W-2, 1099-DIV, K-1, and so on — routes it to the right section of the filing, and processes the whole batch in parallel. There's no need to pre-sort, rename, or label anything.

For very large uploads, processing can take several minutes. You can close the tab and come back; Byron keeps processing in the background.
