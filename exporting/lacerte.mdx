Byron exports to Lacerte through the Data Conductor import system. Byron generates a CSV named after the client number, and Lacerte's Data Conductor reads that CSV into the tax return.

There's a one-time setup to register an import profile in Lacerte. After that, every export is a two-step process: download the CSV from Byron, then import it through Data Conductor.

<Note>
  If you run into problems during setup, reach out to [support@usebyron.com](mailto:support@usebyron.com).
</Note>

## One-time setup: create the BYRON import profile

You only need to do this once per Lacerte installation. The profile tells Data Conductor how to read the CSV files Byron produces.

<Steps>
  <Step title="Open Data Conductor">
    In Lacerte, go to **Import → Data Conductor**.

    <Frame>
      ![lacerte-step1-CI7lpnak](/images/lacerte-step1-CI7lpnak.png)
    </Frame>
  </Step>
  <Step title="Open Profile Manager">
    Click **Profile Manager** in the toolbar, then click **Add** to create a new profile.

    <Frame>
      ![lacerte-step2-ni9-UB9E](/images/lacerte-step2-ni9-UB9E.png)
    </Frame>
  </Step>
  <Step title="Select a sample CSV and file type">
    Select a CSV file that you've already exported from Byron. The file name should be the client number (for example, `12345.csv`) — Data Conductor uses the file name to know which client the data belongs to.

    Choose **Delimited** as the file type, then click **Next**.

    <Frame>
      ![lacerte-step3-DX3D\_\_JH](/images/lacerte-step3-DX3D__JH.png)
    </Frame>
  </Step>
  <Step title="Configure delimiter settings">
    Set the delimiter to **Comma** and the Text Qualifier to `"` (double quote). Click **Next**.

    <Frame>
      ![lacerte-step4-CFqZ-I3b](/images/lacerte-step4-CFqZ-I3b.png)
    </Frame>
  </Step>
  <Step title="Assign categories in exact order">
    Use the arrows to move categories from **Available Categories** into **Assigned Order**. The order must match exactly:

    1. Account Number
    2. Description
    3. Current Year Balance
    4. Prior Year Balance
    5. Series
    6. Prefix
    7. Tax Code

    Click **Next** when complete.

    <Frame>
      ![lacerte-step5-D0XTGryW](/images/lacerte-step5-D0XTGryW.png)
    </Frame>
  </Step>
  <Step title="Name the profile and finish">
    Enter `BYRON` as the **Vendor Profile Name** and `1.0` as the **Version Number**. Click **Finish**.

    <Frame>
      ![lacerte-step6-Cda0wxAU](/images/lacerte-step6-Cda0wxAU.png)
    </Frame>
  </Step>
</Steps>

Setup is done. The BYRON profile will now appear under Custom Profiles for every future import.

## Exporting a filing into Lacerte

## Before you start

Every document you want to export must be in **Approved** status. If anything in the filing is still in review, finish approving it first. See [Review process](/reviewing-data/review-process) for how to approve documents.

Once the initial setup is complete, each export is two parts: download the CSV from Byron, then run it through Data Conductor.

### Part A — download the CSV from Byron

<Steps>
  <Step title="Open the filing and create an export">
    Navigate to the client's filing page and click **Create an Export** in the top-right corner.
  </Step>
  <Step title="Configure the export">
    In the Configure Export dialog:

    - Select **Lacerte** as the destination.
    - Enter the **Client Locator** — this must match the client number in Lacerte.
    - Choose which documents to include.
  </Step>
  <Step title="Initiate the export">
    Click **Initiate Export** at the top or bottom of the dialog.
  </Step>
  <Step title="Download the CSV">
    Click the export file to download it. Save it somewhere you can find from Lacerte.
  </Step>
</Steps>

<Tip>
  Byron names the CSV after the client number you entered. Data Conductor uses that file name to figure out which client to import into, so don't rename the file before importing.
</Tip>

### Part B — import the CSV into Lacerte

<Steps>
  <Step title="Open Data Conductor and start the import">
    In Lacerte, go to **Import → Data Conductor**. Select **Import Other Accounting Package Data** and click **Next**.

    <Frame>
      ![lacerte-import-1-MoM3kdzk](/images/lacerte-import-1-MoM3kdzk.png)
    </Frame>
  </Step>
  <Step title="Select the BYRON profile and your file">
    Under **Custom Profiles**, choose **BYRON** and click **Next**. Browse to and select the CSV you downloaded from Byron.

    <Frame>
      ![lacerte-import-2-Bdv4NhjR](/images/lacerte-import-2-Bdv4NhjR.png)
    </Frame>

    <Frame>
      ![lacerte-import-2b-DMwogTnA](/images/lacerte-import-2b-DMwogTnA.png)
    </Frame>
  </Step>
  <Step title="Import the client">
    Confirm the CSV appears under **Clients to Import**, then click **Finish** to complete the import.

    <Frame>
      ![lacerte-import-3-DJN-TEql](/images/lacerte-import-3-DJN-TEql.png)
    </Frame>
  </Step>
  <Step title="Transfer the data into the return">
    Select the client and click the **Transfer** button to move the data from Data Conductor's Working Trial Balance into the actual Lacerte return.

    <Frame>
      ![lacerte-import-4-jzup66VB](/images/lacerte-import-4-jzup66VB.png)
    </Frame>
  </Step>
</Steps>

<Warning>
  Two things to remember about Data Conductor:

  - **Click Transfer.** The import only loads data into Data Conductor's staging area. Without clicking **Transfer**, nothing reaches the actual Lacerte return.
  - **Don't delete the staging area data after Transfer.** The imported data must remain in Data Conductor's staging area even after a successful Transfer. Deleting it can result in the loss of imported data inside the Lacerte return.
</Warning>

## <Icon icon="bookmark" /> Best practice: make corrections in Byron, not Lacerte

Lacerte preserves any manually entered field values across imports. If you correct a value directly in Lacerte and then re-import the same field from Byron later, both values are kept — and Lacerte will show them as two sub-entries that sum to the wrong total.

For example: if you manually enter an estimated K-1 distribution of \$50 in Lacerte while waiting for the actual K-1, and then later import the actual K-1 from Byron with the value of \$50,000 the field in Lacerte will end up showing \$50,050 with manual entry and the \$50,000 import stacked on top of each other as two sub-entries.

To avoid this:

- Make all corrections inside Byron, where the audit trail and confidence scoring stay intact.
- Treat Lacerte imports as one-time events for each filing where possible.
- If a value needs to change after you've already imported into Lacerte, update it in Byron and re-export — but be aware that any manual edits you've made directly in Lacerte for that field will still be present and may stack with the new import.

The reviewed, approved data in Byron is what should reach Lacerte — not a mix of manual entries and imports.
