Exporting is the last step in a Byron filing. After you've reviewed and approved the extracted data, Byron packages it in the format you need — a direct import for Lacerte or CCH Axcess, an Excel-based file structured for GoSystem, or a standalone Excel workbook or collated PDF you can use however you want. The data that leaves Byron is the reviewed, approved version — corrections applied, audit trail preserved, relationships between forms intact — so the return arrives coherent on the other side.

This page covers the mechanics common to every export: when a document is eligible to be included, how single and batch exports work, and the three-step dialog you'll see every time you click **Create an Export**. For the specifics of each destination, see the per-destination pages linked at the end.

## What Byron exports to

Byron allows you to pick the destination of your export and format.

<AccordionGroup>
  <Accordion title="Tax software exports">
    - **Lacerte** — Byron generates a CSV file that imports directly into Lacerte through Data Conductor. Client IDs are preserved so the data lands on the right return.
    - **CCH Axcess** — Byron generates a `.dat` file that CCH Axcess imports through Batch Manager. CCH supports two routes for getting that file in: a direct API integration that pushes the import without any manual file handling, or a manual file download you import yourself. Both routes use the same underlying file — your firm picks whichever fits its setup.
    - **GoSystem** — Byron generates an Excel-based file structured for use within GoSystem workflows. Unlike Lacerte and CCH Axcess, this is not a direct tax-software import — it's a deliverable you work with inside GoSystem yourself.
  </Accordion>

  <Accordion title="General file downloads">
    - **Excel** — a standalone workbook you can open, review, or hand off to another tool.
    - **PDF** — a collated PDF of the batch, with every source document combined into a single download.

    Use these when you want the data outside any tax-software import — for a workpaper, a partner review, or a client handoff.
  </Accordion>
</AccordionGroup>

## Documents have to be Approved

Make sure the document you want to export has already been reviewed and approved.

For how a document reaches Approved status, see [Review process](/reviewing-data/review-process).

## Single export or batch export

Byron supports both **single-document exports** and **multi-document batches**.

- **Single-document export.** Use this when you've just approved a single document — a late-arriving K-1, a corrected 1099 — and want to ship it by itself without waiting for the rest of the filing.
- **Multi-document batch.** The common case at the end of a filing. You've approved the full client package — W-2s, 1099s, K-1s, whatever else came in — and you send them all at once. You pick the scope by choosing which documents go into the batch at export time.

## Creating an export

From the client's workspace, click **Create an Export**. A dialog opens with three sections that you work through in order.

<Steps>
  <Step title="Step 1: Choose your destination">
    Pick where the documents should go: Lacerte, CCH Axcess, GoSystem, Excel, or PDF.
  </Step>
  <Step title="Step 2: Choose your format">
    Once a destination is selected, the format section opens. Here you pick the format if multiple exist and add the Client ID if applicable.
  </Step>
  <Step title="Step 3: Select your documents">
    The last section lists every Approved document in the filing. Check the ones you want in this batch.
  </Step>
</Steps>

## What travels with the export

Every export carries the same underlying data regardless of destination. The field values that ship are the reviewed, approved ones — not Byron's original extractions before corrections. The tax-software client ID travels with the batch so the data lands on the right return.
