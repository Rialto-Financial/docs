---
title: "How Byron works for business tax"
description: "A step-by-step walkthrough of how Byron's agents move a business tax return from client intake through workpapers, forms, and export."
---

Byron's business tax tool moves through the lifecycle of a return one stage at a time, with agents handling the work and surfacing each step for your team to review and sign off. This page walks through the full flow, starting from a client that's already been created.

<AccordionGroup>
  <Accordion title="1. Intake">
    The first step is asking the client for what's needed to set up the return. From the client workspace, you send an intake request that asks for:

    - Last year's tax return
    - Work papers
    - Last year's tax software file
    - A questionnaire

    The request goes out to the client and waits for them to upload.
  </Accordion>

  <Accordion title="2. Document processing">
    When the client uploads documents, the agent tab opens and Byron processes each document live. You can see what the agent is doing on each one as it works through the batch.
  </Accordion>

  <Accordion title="3. The index">
    Once the documents are processed, Byron populates the index — the full scope of what's going to be in the return based on the information received so far. It includes:

    - **Entity information**
    - **Tax year**
    - **Filing scope** — including which states the return will cover (for example, California, Texas)
    - **Chart of accounts preview** — typically hundreds of lines, available for review
    - **Book-to-tax items** — including depreciation, meals, and bad debt
    - **Other items** relevant to the return

    The index is the team's checkpoint that what Byron has assembled matches the scope of the actual return.
  </Accordion>

  <Accordion title="4. The document manifest">
    After the index is reviewed, Byron's tax engine identifies the documents still needed for the coming year based on everything it's learned about the client to that point. The list appears in the documents view.

    If the manifest looks right, click **Email client**. Byron's agent picks up the request, drafts the email, and surfaces it in the agent tab. You can preview the full email — including all the items being requested — and send it from there. The status of each requested document updates to **Requested** as the email goes out.
  </Accordion>

  <Accordion title="5. Processing the new documents">
    When the client returns the requested documents, Byron processes them the same way — live, in the agent tab, with visibility into where each document is in the pipeline.
  </Accordion>

  <Accordion title="6. Trial balance">
    Once enough financial information is in, Byron generates the trial balance from the financials received. The trial balance maps directly to what you'll see in the Excel export later.

    You can edit values directly in the browser before signing off — corrections happen in place rather than via separate uploads. When the trial balance is final, sign off on the version and move to workpapers.
  </Accordion>

  <Accordion title="7. Workpapers">
    When you're ready to generate workpapers, Byron's agent builds them in the background. The first cut typically includes:

    - **Partner capital accounts** — and how those have changed over the year.
    - **Tax account mappings**
    - **Carry forwards**
    - **Roll forwards**

    Some workpapers may still need to be filled in — they appear blank but in the same structure so it's clear what's needed.

    You can sign off on individual workpapers as they're reviewed, and multiple team members can do this in parallel.
  </Accordion>

  <Accordion title="8. Forms">
    After the workpapers are signed off, Byron produces the filled-in PDFs for the tax return forms. The forms view shows:

    - The list of forms on the left
    - The filled-in PDF in the center
    - Per-form callouts on the right where Byron flags items your team should look at

    You can also have an agent run on any individual callout for a deeper review.
  </Accordion>

  <Accordion title="9. Export">
    The final stage is export. Byron supports two export paths for a business return:

    - **To your tax software** — for direct import into the return module.
    - **To Excel** — the workbook mirrors the workpapers exactly, with two-way sync so edits made in Excel flow back into Byron.
  </Accordion>
</AccordionGroup>

## Audit log

Throughout the entire flow, every action is captured in an audit log — both actions taken by Byron's agents and actions taken by your team. The firm always has a clear record of who did what and when.
