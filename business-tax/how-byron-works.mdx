---
title: "How Byron works for business tax"
description: "A step-by-step walkthrough of how Byron's agents move a business tax return from client intake through workpapers, forms, and export."
---

Byron's business tax tool moves through the lifecycle of a return one stage at a time, with agents handling the work and surfacing each step for your team to review and sign off. This page walks through the full flow.

<AccordionGroup>
  <Accordion title="1. Setup">
    The first stage depends on whether this is a new client or a returning client.

    **For a returning client (roll forward).** If you worked with the client last year, Byron rolls forward from the prior year baseline. Behind the scenes, Byron reads the prior year work paper and the PDF tax return to derive the starting point for this year's return. The result is an index already populated with key facts (entity information, filing scope, and so on) and an initial document request list built from the same information you needed last year — including data from the prior year tax software file (for example, a CCH `.dat` file) where available.

    **For a new client (intake).** From the client workspace, you send an intake request that asks for:

    - Last year's tax return
    - Work papers
    - Last year's tax software file
    - A questionnaire

    The request goes out to the client and waits for them to upload.
  </Accordion>

  <Accordion title="2. Document processing">
    When documents are uploaded — whether through the client upload portal or directly within Byron — the agent tab opens and Byron processes each document live. You can see what the agent is doing on each one as it works through the batch.
  </Accordion>

  <Accordion title="3. The index">
    Once documents are processed, Byron populates the index — the full scope of what's going to be in the return based on the information received so far. It includes:

    - **Entity information**
    - **Tax year**
    - **Filing scope** — including which states the return will cover (for example, California, Texas)
    - **Chart of accounts preview** — typically hundreds of lines, available for review
    - **Book-to-tax items** — including depreciation, meals, and bad debt
    - **Other items** relevant to the return

    The index is the team's checkpoint that what Byron has assembled matches the scope of the actual return.

    **Editing the index directly.** You can update the index in place if there's something you know needs to be there for this year. For example, if you're expecting a new book-to-tax item this year — say, officer life insurance — you can add it to the index without waiting for it to be inferred from a document.
  </Accordion>

  <Accordion title="4. The document manifest">
    Alongside the index, Byron's tax engine produces a list of documents still needed for the coming year, drawing from the prior year work paper, the prior year tax software file, and any roll-forward context for a returning client.

    **Adding requests manually.** If there's a source document request you want to add that isn't on the list, you can search for the document type and add it manually — it joins the rest of the request list.

    **Removing a request.** If a request was added in error, is a duplicate, or is no longer needed, click the trash icon on the placeholder row to remove it. If a document was already linked to that request, the document stays on the project — only its link to the request is cleared.

    **Sending the request.** When the manifest looks right, click **Email client**. Byron's agent picks up the request, drafts the email — listing every item being requested — and surfaces it in the agent tab for your review. The email also includes a link to the Byron upload portal, where the client can drop in the documents directly. You preview the email and send it from there. The status of each requested document updates to **Requested** as the email goes out.
  </Accordion>

  <Accordion title="5. Processing new documents">
    When the client returns the requested documents — through the upload portal or otherwise — Byron processes them the same way as the initial batch: live, in the agent tab, with visibility into where each document is in the pipeline.

    **Ad-hoc uploads.** If the client uploads something outside the request list — for example, a forgotten questionnaire — Byron processes it and updates both the index and the document request list to reflect what's now been received.
  </Accordion>

  <Accordion title="6. Trial balance">
    Once enough financial information is in, Byron generates the trial balance from the financials received. The trial balance maps directly to what you'll see in the Excel export later.

    You can edit values directly in the browser before signing off — corrections happen in place rather than via separate uploads. When the trial balance is final, sign off on the version and move to workpapers.
  </Accordion>

  <Accordion title="7. Workpapers">
    When you're ready to generate workpapers, Byron's agent builds them in the background. The first cut typically includes:

    - **Partner capital accounts** — and how those have changed over the year.
    - **Tax account mappings**
    - **Carry forwards**
    - **Roll forwards**

    Some workpapers may still need to be filled in — they appear blank but in the same structure so it's clear what's needed.

    You can sign off on individual workpapers as they're reviewed, and multiple team members can do this in parallel.
  </Accordion>

  <Accordion title="8. Forms">
    After the workpapers are signed off, Byron produces the filled-in PDFs for the tax return forms. The forms view shows:

    - The list of forms on the left
    - The filled-in PDF in the center
    - Per-form callouts on the right where Byron flags items your team should look at

    You can also have an agent run on any individual callout for a deeper review.
  </Accordion>

  <Accordion title="9. Export">
    The final stage is export. Byron supports two export paths for a business return:

    - **To your tax software** — for direct import into the return module.
    - **To Excel** — the workbook mirrors the workpapers exactly, with two-way sync so edits made in Excel flow back into Byron.
  </Accordion>
</AccordionGroup>

## Audit log

Throughout the entire flow, every action is captured in an audit log — both actions taken by Byron's agents and actions taken by your team. The firm always has a clear record of who did what and when.
