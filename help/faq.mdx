## General questions

<AccordionGroup>
  <Accordion title="What is Byron?">
    Byron is an AI-powered tax document processing platform that automates the extraction of data from tax forms with 98–99% accuracy. Byron integrates directly with CCH Axcess, Lacerte, and GoSystem, or exports to Excel, eliminating hours of manual data entry for accounting firms.

    Byron processes complex documents including K-1 packages (with K-2, K-3, and state forms), W-2s, 1099s, and other tax forms, with a human-in-the-loop review workflow that confirms accuracy before export.
  </Accordion>

  <Accordion title="How does Byron differ from other tax automation tools?">
    Byron is built specifically for the complex needs of modern accounting firms:

    - **Higher accuracy.** Byron's agent reaches 98–99% extraction accuracy, compared to 75–85% for typical competing AI solutions.
    - **Complex K-1 support.** Byron handles 120\+ page K-1 packages including K-2, K-3, and all state forms.
    - **Field-level confidence.** Every field is sorted into High Confidence or Review, so you know which fields need closer attention.
    - **Direct integrations.** Export to CCH Axcess, Lacerte, or GoSystem; download Excel workbooks or collated PDFs.
    - **Built by CPAs.** The team includes experienced tax professionals who understand the workflow.
  </Accordion>

  <Accordion title="Do I need to install any software?">
    No. Byron is a web-based platform accessible from any modern browser. You log in at your assigned URL — no installation required.

    The one exception is if you use Lacerte and want to import data through Data Conductor, which requires a one-time configuration inside Lacerte. See [Export to Lacerte](/exporting/lacerte) for details.
  </Accordion>

  <Accordion title="How long does it take to process documents?">
    Processing times vary by document complexity:

    - **Simple forms (W-2, 1099):** 15–30 seconds
    - **Standard K-1 packages:** 30–60 seconds
    - **Complex K-1 packages (100\+ pages):** 2–3 minutes

    Documents process asynchronously, so you can keep working while Byron handles them in the background. The filing workspace updates as documents move into Ready for Review.
  </Accordion>
</AccordionGroup>

## Accuracy and processing

<AccordionGroup>
  <Accordion title='What does "98–99% extraction accuracy" mean?'>
    Extraction accuracy is the percentage of fields that come out correctly without requiring manual correction, measured across all field types (text, numbers, dates, checkboxes) on every form Byron supports.

    For context, manual data entry by humans typically achieves 90–97% accuracy, and competing AI solutions typically achieve 75–85%.

    Even at 98–99%, Byron always recommends human review before export. The review workflow is designed to make that quick — high-confidence fields can be skimmed, while lower-confidence fields are surfaced for closer attention. See [Review screen](/reviewing-data/review-screen) for how Byron decides what to flag.
  </Accordion>

  <Accordion title="What happens if the agent makes a mistake?">
    The review workflow is designed to catch and correct extraction errors:

    1. **Confidence buckets.** Fields are sorted into High Confidence, Review, and Corrected buckets to guide your attention.
    2. **Side-by-side review.** The source document sits next to the extracted data, with anchors that take you to the exact source region for each field.
    3. **Inline corrections.** Click any field to edit the value. You can add a comment explaining the change.
    4. **Audit log.** Every correction is logged with who made the change, when, and why — used both for your firm's review trail and to improve the agent over time.
  </Accordion>

  <Accordion title="Can Byron handle poor-quality scans or handwritten documents?">
    Yes, but confidence drops on harder source material. Byron is trained to handle:

    - Slightly blurry or low-resolution scans (200 DPI minimum recommended)
    - Skewed or rotated pages
    - Documents with annotations or stamps
    - Some handwritten numbers and text on otherwise printed forms

    For best results, upload clear scans at 300 DPI or higher. If the source is very poor quality, the agent flags the affected fields for manual review.
  </Accordion>

  <Accordion title="Does Byron read footnotes and attachments on K-1s?">
    Yes. K-1 processing includes:

    - **Footnote analysis.** Byron extracts and interprets footnotes that affect field values.
    - **Code parsing.** All Box 20 codes and their amounts are extracted.
    - **K-3 forms.** Complete extraction of foreign tax information.
    - **State forms.** All state K-1 variants are processed.
    - **Attachment summaries.** Key information from attached statements is captured.

    This is what makes K-1 packages work end-to-end — critical information buried in footnotes or supplemental schedules doesn't get missed.
  </Accordion>
</AccordionGroup>

## Integrations

<AccordionGroup>
  <Accordion title="Which tax software does Byron integrate with?">
    Direct integrations available today:

    - **CCH Axcess Tax** — `.dat` file imported through Return Manager, with a direct API integration available so the file pushes into CCH on its own.
    - **Lacerte Tax** — CSV export for Data Conductor import.
    - **GoSystem Tax** — Excel-based file structured for use within GoSystem workflows. Unlike Lacerte and CCH Axcess, this is not a direct tax-software import — it's a deliverable you work with inside GoSystem yourself.

    File-based options for any tax software:

    - **Excel** — full workpaper workbook with per-document detail and an aggregated K-1 view.
    - **Collated PDF** — every source document in the batch as a single download.

    If your tax software isn't listed, let us know — Byron adds integrations based on customer demand.
  </Accordion>

  <Accordion title="Can Byron integrate with my practice management software?">
    Byron supports integrations with practice management and document collection tools so client records and documents flow into the platform automatically. Specific integrations available for your firm vary — contact the team to confirm what's live for your account and what's on the integration roadmap.
  </Accordion>

  <Accordion title="How does the CCH Axcess integration work?">
    The CCH Axcess integration is built around a `.dat` file. There are two ways to get that file into CCH Axcess: through Byron's direct API integration with CCH (no manual file handling), or by downloading the `.dat` file from Byron and importing it through Return Manager yourself. Both methods use the same underlying file — your firm picks whichever fits your setup.

    The setup in Byron is the same either way:

    1. In Byron, click **Create an Export** from the client workspace.
    2. Choose **CCH Axcess** as the destination, then choose your format and enter your CCH Client ID.
    3. Select the approved documents to include and click **Initiate Export**. Byron generates the `.dat` file.
    4. If your firm is on the API integration, Byron pushes the file into CCH Axcess on its own. If you're on manual import, download the file and import it into the matching client in CCH Axcess Return Manager.

    See [Export to CCH Axcess](/exporting/cch-axcess) for the detailed step-by-step.
  </Accordion>

  <Accordion title="Is my data secure when integrating with CCH or Lacerte?">
    Yes. Security measures include:

    - **OAuth 2.0 authentication.** Secure, token-based access — no passwords stored.
    - **Encrypted transmission.** All data transfers use TLS 1.3 encryption.
    - **Minimal permissions.** Byron only requests permissions needed for import and export.
    - **No persistent export storage.** Export files are generated on demand and not stored permanently.
    - **Audit logging.** All integration activity is logged for compliance.

    Byron's security practices align with IRS Publication 4557 and industry standards for tax data protection.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title='My document is stuck in "Processing" status. What should I do?'>
    Processing times vary by document complexity, but if a document has been processing for more than five minutes:

    1. Refresh your browser to check whether the status has updated.
    2. Check whether the document is unusually large (100\+ pages take longer).
    3. Look for any error message on the document status panel.
    4. If still stuck after ten minutes, contact support with the document ID.

    Common causes of delayed processing include very large file sizes (10\+ MB PDFs), complex K-1 packages with extensive footnotes, and corrupted or password-protected PDFs.
  </Accordion>

  <Accordion title="The agent extracted incorrect data. How do I fix it?">
    To correct extraction errors:

    1. Open the document on the review screen.
    2. Click on the incorrect field.
    3. Enter the correct value.
    4. Optionally add a comment explaining the change.
    5. Approve the document to release the corrections into the filing.

    Corrections show up as `Corrected` status chips on the relevant fields. The audit log tracks who made each change, when, and why — so a senior reviewer can always see what was edited and why.

    See [Review process](/reviewing-data/review-process) for the full review workflow.
  </Accordion>

  <Accordion title="My CCH Axcess import failed. How do I troubleshoot?">
    Common CCH import issues and solutions:

    - **Client ID mismatch.** Verify the CCH client ID attached to the filing in Byron matches your CCH client exactly.
    - **Validation errors.** Review the validation step in CCH for specific field issues.
    - **`.dat` file not recognized.** Ensure the file wasn't corrupted during download.
    - **Missing data after import.** Confirm you completed every step in the CCH import wizard.

    For persistent issues, contact support with your export details and the team can help troubleshoot.
  </Accordion>

  <Accordion title="Documents from my document collection integration aren't syncing. Why?">
    Check these common issues:

    1. **Folder mapping.** Verify folders are correctly mapped in your integration settings.
    2. **Upload completion.** Ensure files have finished uploading on the source side, not still syncing.
    3. **File format.** Confirm files are in supported formats.
    4. **Connection status.** Check that the integration shows Connected.
    5. **Sync frequency.** Wait for the next sync cycle, or trigger a manual sync from the integration settings.
  </Accordion>
</AccordionGroup>
